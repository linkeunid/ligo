# Ligo 1.0 Release Roadmap

## Current Status: 1.0 ✅ Ready for Release

Ligo has completed all 1.0 requirements:
- Complete DI system (Singleton, Transient, Request-Scoped)
- Full HTTP routing with Guards, Pipes, Interceptors, Exception Filters
- Module-level lifecycle hooks
- Dynamic modules
- Built-in utilities (guards, pipes, interceptors, response helpers)
- 211 tests passing (47.3% coverage)
- Integration tests for full app lifecycle
- Performance benchmarks
- Comprehensive documentation (API, migration, best practices, stability, performance, deployment)
- Stability policy with semver guarantees
- Deployment guides for Docker, Kubernetes, and cloud platforms

## Out of Scope (By Design)

- ❌ Runtime lazy loading - Not idiomatic in Go
- ❌ Decorators - Go doesn't have decorators; uses builder pattern

## Future Work (Post-1.0)

### Microservices Package

The next major package will be `github.com/linkeunid/ligo/microservices` for building distributed systems.

**Implementation Guide:** See [Microservices Documentation](../microservices.md)
**Package Development Guide:** See [Package Development Guide](../package-development.md)

**Planned Features:**
- TCP transport for direct messaging
- Redis transport (pub/sub, RPC)
- RabbitMQ transport (queues, exchanges)
- Message patterns: Request/Response, Pub/Sub, Event-driven
- Client and Server abstractions

### Database Integration

Separate packages for database integrations (similar to `@nestjs/typeorm`):
- `github.com/linkeunid/ligo/database` - Core database interfaces
- `github.com/linkeunid/ligo/database/pgx` - PostgreSQL driver
- `github.com/linkeunid/ligo/database/gorm` - GORM integration
